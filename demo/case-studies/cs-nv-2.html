Case Study

Inference Optimization at Edge

01Overview

A manufacturing client needed to deploy quality control vision models directly on factory-floor edge devices with limited compute budgets and no reliable cloud connectivity. Models trained in the cloud needed to run efficiently on embedded GPU hardware.

02Problem

Production models required 8GB of GPU memory and 200ms inference latency, far exceeding the 2GB memory budget and 30ms latency requirement of the edge devices. Initial attempts to deploy directly resulted in out-of-memory crashes.

03Solution

Models were compressed through structured pruning, quantised to INT8 precision, and compiled using a hardware-specific inference optimiser. Knowledge distillation was used to train smaller student models that matched the accuracy of the original while fitting within the hardware constraints.

04Results

12ms

Inference latency (target: 30ms)

1.4GB

Model memory footprint

98.7%

Accuracy retained post-compression

The compressed models exceed the performance targets while maintaining near-identical accuracy. The factory now operates a fully offline quality control system with zero cloud dependency.
